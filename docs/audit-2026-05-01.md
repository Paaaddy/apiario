## Critical — Test coverage gaps

Error-recovery paths in production code with zero test coverage.

### T-1 · `loadProfile` corrupted localStorage
**File:** `src/hooks/useProfile.js:60`
`loadProfile()` wraps `JSON.parse` in try/catch and returns `DEFAULT_PROFILE` on failure. No test verifies this path.
```js
it('returns DEFAULT_PROFILE on corrupted localStorage', () => {
  localStorage.setItem('apiario-profile', '{bad json')
  const { result } = renderHook(() => useProfile())
  expect(result.current.profile).toEqual(expect.objectContaining({ onboardingDone: false }))
})
```

### T-2 · `loadLog` corrupted localStorage
**File:** `src/hooks/useTaskLog.js:7`
`loadLog()` returns `[]` on JSON parse failure. No test covers this path.
```js
it('returns empty log on corrupted localStorage', () => {
  localStorage.setItem('apiario-log', '{bad json')
  const { result } = renderHook(() => useTaskLog())
  expect(result.current.log).toEqual([])
})
```

### T-3 · `MAX_ENTRIES` cap enforcement
**File:** `src/hooks/useTaskLog.js:5`
`cap()` slices the log to 500 entries. No test verifies the cap triggers or that entries beyond 500 are discarded.
```js
it('caps log at MAX_ENTRIES (500) when adding entries', () => {
  // seed 501 entries, verify log.length === 500
})
```

## Informational — Performance

### P-1 · `logByTaskId` Map rebuilt every render
**File:** `src/screens/SeasonScreen.jsx:31`
Derived from `log` but has no `useMemo` guard. Cost scales with log size.
Fix: `const logByTaskId = useMemo(() => new Map(log.map(...)), [log])`

### P-2 · `t` callback recreated every render
**File:** `src/context/LanguageContext.jsx:29`
`t` recreated on every `LanguageProvider` render, causing unnecessary re-renders in all consumers.
Fix: `useCallback([locale])`
